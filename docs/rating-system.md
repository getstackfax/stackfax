Stackfax Rating System

Stackfax uses a 1–5 star rating system to make Ai stacks easy to understand, compare, and improve.

The star rating is not just about raw power.

A strong stack should match the user’s actual use case, workload, budget, safety needs, privacy level, permission boundaries, and technical skill.

⸻

Rating Principle

A 5-star stack is not always the biggest or most expensive stack.

A 5-star stack is the stack that best fits the job.

Core rule:

Do not reward complexity by default. Reward fit.

⸻

⭐ 1-Star Stack

A risky or poorly matched stack.

Common Signs

* expensive models used for simple tasks
* no clear use case
* no defined workflow
* no approval gates
* unclear permissions
* high privacy risk
* unnecessary hardware purchases
* no cost controls
* agents can touch important systems too easily
* credentials are loosely handled
* customer-facing actions may happen without review
* no run receipts or evidence of what happened

Verdict

Do not scale this stack yet.

Simplify it, reduce access, define the use case, and add approval gates before giving agents more responsibility.

Typical Risk Flags

* Overbuild Risk
* Token Burn Risk
* Human Approval Required
* Credential Isolation Risk
* Customer Data Risk
* Production Not Ready

⸻

⭐⭐ 2-Star Stack

A usable but messy stack.

Common Signs

* some useful tools or models are present
* cost control is weak
* model routing is unclear
* safety rules are incomplete
* hardware may be overkill
* the user may not understand what agents can access
* context is loaded too broadly
* subscriptions may overlap
* the first workflow is not fully defined
* run receipts are missing or incomplete

Verdict

This stack can be tested, but it needs cleanup before real business, financial, production, or customer-facing use.

Typical Risk Flags

* Model Routing Needed
* Subscription Overlap Risk
* Context Bloat Risk
* Approval Gates Missing
* Cost Visibility Missing
* Tool Loop Burn

⸻

⭐⭐⭐ 3-Star Stack

A solid starter stack.

Common Signs

* use case is mostly clear
* basic safety rules exist
* hardware choice is reasonable
* cost risk is manageable
* some approval gates are present
* good enough for learning and controlled testing
* local vs cloud direction is mostly sensible
* obvious risky actions are not fully automated
* next improvement path is visible

Verdict

Good starting point.

Improve model routing, documentation, run receipts, and approval gates before scaling.

Typical Risk Flags

* Recheck Needed
* Human Approval Required
* Token Burn Risk
* Workflow Fit Needs Clarification
* Version Drift Risk

⸻

⭐⭐⭐⭐ 4-Star Stack

A strong stack.

Common Signs

* clear use-case fit
* good model routing
* reasonable cost controls
* strong approval gates
* privacy risks are understood
* hardware matches the workload
* human review exists for important actions
* setup is explainable to the user
* workflow has clear inputs and outputs
* permissions are scoped
* run receipts or logs exist
* upgrade path is staged

Verdict

This stack is ready for serious use with proper monitoring, rechecks, and operating discipline.

Typical Badges

* StackChecked
* Token-Smart
* Cloud-First
* Local-Ready
* Human Approval Required
* Run Receipts Required

⸻

⭐⭐⭐⭐⭐ 5-Star Stack

An excellent stack.

Common Signs

* cost-aware
* use-case matched
* well-routed
* well-documented
* safe by default
* clear permission boundaries
* strong human approval rules
* hardware and hosting are justified
* scalable without becoming reckless
* easy to audit, explain, and improve
* run receipts are clear
* version drift is handled with rechecks
* sensitive data is isolated
* failure paths are understood
* every paid tool has a job

Verdict

This is a high-quality stack.

It is not just powerful.

It is controlled, efficient, explainable, and matched to the job.

Typical Badges

* StackChecked Fresh
* Token-Smart
* Strong Workflow Fit
* Approval Gates Present
* Cost Visibility Present
* Run Receipt Layer Strong
* Recheck Plan Present

⸻

Star Rating Bands

Suggested mapping from Stack Score to star rating:

* ⭐: 0–24
* ⭐⭐: 25–49
* ⭐⭐⭐: 50–69
* ⭐⭐⭐⭐: 70–84
* ⭐⭐⭐⭐⭐: 85–100

The number should be explained in plain language.

Do not give stars without a verdict.

⸻

What Should Raise A Rating

A rating should improve when the stack has:

* clear use case
* clear workflow
* model routing
* cost controls
* relevant context only
* approval gates
* scoped permissions
* credential isolation
* local/cloud fit
* realistic hardware choice
* run receipts
* fallback plan
* 30-day recheck plan
* safe upgrade path

⸻

What Should Lower A Rating

A rating should drop when the stack has:

* unclear use case
* tool pile without workflow
* premium model overuse
* hidden token burn
* no approval gates
* broad credential access
* customer data risk
* fragile UI automation
* no cost visibility
* no run receipts
* overbuilt hardware
* no fallback path
* production use without testing
* version drift ignored

⸻

Core Stackfax Rule

Cheap model drafts.

Strong model decides.

Human approves anything touching customers, money, inventory, credentials, private files, public posting, or production systems.

Use code for extraction.

Use Ai for judgment.

The model is not the stack.

The workflow is the moat.

⸻

Final Rule

A star rating should make the decision easier.

If the user sees the rating but still does not know what to fix next, the rating failed.
