# OpenClaw Rabble Scout Plan

This file defines how Stackfax may eventually use OpenClaw-style agents as Rabble Scouts.

Rabble Scouts are not trading bots, posting bots, or autonomous business operators.

They are observation agents.

Their job is to scout, summarize, classify, and report.

## Core Idea

Rabble Scouts help Stackfax detect public Ai stack confusion.

They watch for repeated patterns around:

- OpenClaw setup confusion
- Mac mini and local hardware questions
- Token burn complaints
- Model subscription overlap
- Vendor stack announcements
- Creator stack guides
- Agent permission mistakes
- Credential isolation risk
- Business automation safety questions

The goal is to improve Stackfax reports, templates, badges, and education.

## What Rabble Scouts Can Do

Rabble Scouts may eventually help with:

- Finding public posts or discussions
- Summarizing public user pain points
- Classifying examples by user type
- Flagging repeated stack confusion
- Suggesting possible Stackfax report types
- Suggesting possible badges or risk flags
- Drafting field research notes
- Drafting internal summaries
- Preparing human-review queues

## What Rabble Scouts Cannot Do

Rabble Scouts should not:

- Post publicly
- Reply to users automatically
- Send messages
- Contact customers
- Touch credentials
- Access wallets
- Access financial accounts
- Access private files
- Access private messages
- Modify GitHub files without approval
- Change website content without approval
- Make purchases
- Run business actions
- Pretend to be a human

## Approval Rule

Rabble Scouts can observe and draft.

A human approves before anything is published, posted, committed, sent, or acted on.

Core rule:

Observe first.

Draft second.

Approve third.

Automate last.

## Public-Only Research Rule

Rabble Scouts should only use public sources unless a human explicitly provides approved material.

Allowed public sources may include:

- Public Reddit threads
- Public YouTube videos or comments
- Public X posts
- Public GitHub repos
- Public blogs
- Public product announcements
- Public vendor docs
- Public forums

Not allowed by default:

- Private emails
- Private files
- Private DMs
- Private customer records
- Personal accounts
- Password managers
- Wallets
- Financial accounts
- Client data

## Field Research Output

A Rabble Scout should produce structured notes.

Example output:

- Date found
- Source type
- Public topic
- User type
- Observed pain
- Stackfax could solve
- Possible report type
- Possible badge or risk flag
- Product lesson
- Useful wording

## Safe First Rabble Scout

The first safe Rabble Scout should be very limited.

It should:

- Read only public examples provided by the user
- Summarize the pattern
- Classify the user type
- Suggest what Stackfax could solve
- Suggest a possible report type
- Suggest a possible risk flag
- Ask before saving or committing anything

It should not browse freely, post, message, buy, sell, trade, or access private accounts.

## OpenClaw Fit

OpenClaw may fit Stackfax later because Stackfax itself is about checking stacks.

But Stackfax should build the grocery list before making the sandwich.

Internal rule:

Stackfax and OpenClaw are PB&J.

We just have to go to the grocery store first.

## Future Agent Roles

Possible future Rabble Scout roles:

### Field Scout

Finds public examples of Ai stack confusion.

### Token Scout

Looks for token burn complaints and cost confusion.

### Hardware Scout

Looks for Mac mini, local model, cloud server, and local Ai box questions.

### Vendor Scout

Tracks new vendor stack announcements and packaged setups.

### Subscription Scout

Tracks model subscription comparison and overlap confusion.

### Business Safety Scout

Tracks business automation risk examples.

## Final Rule

Rabble Scouts do not act.

They observe, summarize, classify, and report.

Stackfax stays human-approved until the system proves it deserves more trust.
