# Stackfax Public Private Roadmap

This file defines how Stackfax should separate the public trust layer from the private business engine over time.

The goal is to build trust publicly while protecting customer data, paid workflows, internal operations, and future automation systems.

## Core Strategy

Public trust layer.

Private business engine.

Stackfax should show enough to build trust and explain the standard.

Stackfax should keep enough private to build the business safely.

## Why This Matters

Stackfax is not just a code project.

It includes:

- Product positioning
- Report logic
- Scoring language
- Badge language
- Templates
- Marketing copy
- Customer workflows
- Future payment flow
- Future intake flow
- Future scoring engine
- Future OpenClaw and Rabble Scout operations

Some of that can be public.

Some of that should stay private.

## Public Layer

The public layer helps users understand what Stackfax is and why it exists.

The public layer can include:

- README
- Brand docs
- Field guide
- Sample reports
- Public report types
- Basic templates
- Marketing copy
- SEO keyword map
- Public safety rules
- Risk category docs
- Report ladder
- Community standards plan
- License strategy
- Public field research patterns

## Public Layer Goals

The public layer should:

- Build trust
- Explain the category
- Help users understand Ai stacks
- Show report examples
- Show safety thinking
- Help with SEO
- Attract early users
- Make Stackfax feel real
- Create shared language around stacks

## Private Layer

The private layer protects real operations.

The private layer should include:

- Real customer submissions
- Real customer reports
- Payment records
- Customer emails
- Support messages
- Report delivery links
- Internal report notes
- Internal scoring logic
- Advanced templates
- Automation scripts
- Private analytics
- Real feedback records
- OpenClaw/Rabble Scout working system

## Private Layer Goals

The private layer should:

- Protect customers
- Protect sensitive data
- Protect paid workflows
- Protect the future scoring engine
- Protect business operations
- Protect internal automation
- Reduce cloning risk
- Keep the public repo clean

## What Should Never Go Public

Do not put these in the public repo:

- Passwords
- API keys
- Recovery codes
- Payment details
- Wallet information
- Private keys
- Customer submissions
- Customer emails
- Real customer reports
- Private business data
- Client data
- Private screenshots
- Internal report delivery links
- Private OpenClaw configs
- Private Rabble Scout outputs
- Internal automation credentials

## Future OpenClaw And Rabble Scout Layer

OpenClaw and Rabble Scouts should stay private at first.

Rabble Scouts may eventually help with:

- Public field research
- Pattern detection
- Stack confusion classification
- Report type suggestions
- Badge suggestions
- Drafting internal research notes

Rabble Scouts should not:

- Post publicly
- Contact users
- Access customer data
- Access credentials
- Modify files without approval
- Touch payments
- Touch wallets
- Touch production systems

## Public Release Rule

Before something goes public, ask:

- Does this build trust?
- Does this expose customer data?
- Does this expose business operations?
- Does this reveal too much of the paid engine?
- Does this help users understand Stackfax?
- Does this create unnecessary cloning risk?
- Does this need to be public right now?

## Private Storage Rule

Real customer work should live outside the public repo.

Possible private storage:

- Private Google Drive
- Private Notion
- Private Airtable
- Private encrypted folder
- Private database later

Early recommendation:

Use a simple private folder and avoid storing sensitive information unless needed.

## Mixed Structure

Best long-term structure:

Public:

- Education
- Public standards
- Sample reports
- Basic templates
- Marketing pages
- Trust rules
- High-level scoring philosophy

Private:

- Customer reports
- Payment operations
- Intake submissions
- Internal scoring tools
- Advanced report templates
- Automation scripts
- OpenClaw/Rabble Scout working system

## Stackfax Principle

Show enough to build trust.

Keep enough private to build the business.

Public trust layer.

Private business engine.
