# Stackfax Private Folder Setup Plan

This file defines the simple private folder structure for early Stackfax report requests.

The goal is to keep real customer submissions, reports, notes, and delivery records out of the public GitHub repo.

## Core Goal

Create one private place for real Stackfax customer work.

The public repo is for public docs, sample reports, templates, and product thinking.

The private folder is for real submissions, real reports, delivery notes, and customer workflow records.

## Recommended First Setup

Use a private Google Drive folder.

Suggested folder name:

Stackfax Private Operations

Why:

- Simple to create
- Easy to organize
- Easy to keep private
- Works with Google Forms
- Can hold docs, reports, exports, notes, and screenshots if needed

## Folder Safety Rule

Do not make the private operations folder public.

Do not share the folder broadly.

Do not store passwords, API keys, wallet information, payment details, private keys, recovery codes, or sensitive personal data.

Stackfax should collect clarity, not secrets.
