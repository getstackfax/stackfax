Stackfax Google Form Build Checklist

This checklist turns the Stackfax report intake layout into a real Google Form.

The goal is to create one simple form that can collect early Free Mini Report and Quick Report requests without asking users for sensitive information.

⸻

Core Goal

Build a simple Stackfax intake form that helps users describe their Ai stack, workflow, tools, goals, risks, and concerns.

The form should collect clarity, not secrets.

Core rule:

Simple beats fancy. One safe intake form is better than five confusing forms.

⸻

Form Name

Stackfax Report Intake

⸻

Form Description

Submit your Ai stack, workflow, tools, and concerns so Stackfax can prepare a report.

Do not submit passwords, API keys, payment details, wallet information, private customer data, private legal documents, private financial documents, private medical documents, private inbox exports, source code secrets, production credentials, or sensitive personal information.

Stackfax only needs a description of your stack, workflow, tools, goals, risks, and concerns.

⸻

Before Building

Confirm you have:

* a Google account for Stackfax
* access to Google Forms
* a safe place to receive responses
* contact email ready
* private folder plan for real submissions
* report delivery workflow
* safety language ready
* intake layout file available

Reference file:

forms/report-intake-google-form-layout.md

If that file does not exist yet, use this checklist as the first live form source.

⸻

Recommended Settings

In Google Forms settings:

* Collect email addresses: On
* Send responders a copy of their response: Optional
* Limit to one response: Off for now
* Allow response editing: Off for now
* Show progress bar: On
* Shuffle question order: Off
* Make this a quiz: Off

Recommended early choice:

* Keep the form simple.
* Do not require Google sign-in unless spam becomes a problem.
* Do not enable file uploads yet.

File uploads can create private-data risk too early.

Confirmation Message

Set the form confirmation message to:

Thanks for submitting your Stackfax intake.

Your stack details have been received.

If this is a paid Quick Report, Stackfax will review the intake and may follow up if more context is needed.

Expected delivery target for accepted Quick Reports:

24 to 48 hours.

⸻

Response Destination

Recommended early setup:

* Store responses in Google Forms / linked Google Sheet.
* Keep the response sheet private.
* Do not make the response sheet public.
* Do not copy real customer details into the public GitHub repo.
* Save only anonymized product lessons publicly.

Suggested private sheet columns should include:

* timestamp
* email
* name or handle
* report type
* main goal
* current stack
* workflow
* cost concern
* risk concern
* automation level
* desired output
* status
* follow-up needed
* report delivered
* feedback received

⸻

Test Before Sharing

Submit one test response.

Check:

* response is received
* email is captured
* required questions work
* safety confirmation works
* long answers are readable
* checkboxes are easy to interpret
* confirmation message is clear
* form does not ask for secrets
* linked sheet is private
* owner/operator can find the response quickly

⸻

Link Handling

Before sharing publicly:

* copy the public form link
* save it in a private note
* add it to Gumroad later
* add it to Stackfax.com later
* add it to future soft launch posts only when ready
* test the form link in a private/incognito browser window

Do not post the form widely until the delivery workflow is ready.

⸻

First Live Use

For the first few real submissions:

* review manually
* check for unsafe private information
* decide whether Free Mini Report or Quick Report fits
* ask follow-up questions if needed
* deliver by email
* log anonymized product lesson only
* update templates if repeated confusion appears

⸻

Final Rule

Simple beats fancy.

One safe intake form is better than five confusing forms.

Collect clarity, not secrets.
