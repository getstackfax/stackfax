## Core Doctrine

- A bigger stack is not always a better stack.
- Do not copy the biggest stack. Find the right stack.
- The model is not the stack.
- A model can win the benchmark and still lose the workflow.
- Token burn is not bad by itself. Misrouted token burn is the leak.
- Local-first is not the same as safe-by-default.
- Agent memory is not agent permission.
- Agents can propose outcomes. Systems verify outcomes.
- A production agent needs a run receipt, not just logs.
- Useful output matters more than agent activity.
- Process first. Automation second.
- Drafting is not execution.
- A stack is not a pile of tools. A stack is the coordination layer between them.
- Pass state, not history.
- Prompt logic without metadata becomes invisible infrastructure.
- A serious Ai stack treats prompts like code: versioned, parameterized, logged, and reviewable.

## Rabble Scout Rule

Rabble Scouts do not act.

They observe, summarize, classify, and report.

Default sequence:

1. Observe.
2. Classify.
3. Distill.
4. Human review.
5. Project source if canon.
6. Build into Stackfax only after approval.

Core scouting rule:

Gold becomes build material. Silver becomes watchlist. Bronze becomes market texture.
