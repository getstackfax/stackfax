Trust Rule

Stackfax is a trust product.

Real testimonials should only be used with clear permission.

Example reactions should always be clearly labeled.

Do not blur the line between fictional examples and real customer feedback.

Core rule:

Example copy can show the kind of moment Stackfax is built for. It should not pretend the moment already happened.

⸻

Allowed Labels

Use clear labels such as:

* Example user reaction
* Sample scenario
* Prototype testimonial
* Built for moments like this
* Fictionalized example
* Example only

Avoid labels that imply real customer proof unless the quote is real and approved.

Do not use:

* Customer testimonial
* Real customer result
* Verified user quote
* Case study
* Customer story

unless the user clearly gave permission.

Real Testimonial Rule

If a real user gives feedback and Stackfax wants to use it publicly, confirm:

* what exact quote can be used
* what name or handle can be shown
* whether company name can be shown
* whether the quote can be lightly edited for clarity
* whether the report type can be referenced
* whether the user wants to stay anonymous

No public testimonial without clear permission.

⸻

Final Rule

Example reactions are useful for explaining the problem.

They are not proof.

Use them to clarify the moment Stackfax is built for.

Do not use them to fake trust.
